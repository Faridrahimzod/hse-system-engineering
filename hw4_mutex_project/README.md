# Mutex

## Описание

В данной работе реализован собственный `Mutex` на C++.

Мьютекс — это примитив синхронизации, который обеспечивает взаимное исключение: в каждый момент времени только один поток может находиться внутри критической секции.

В отличие от спинлока, мьютекс при конкуренции не заставляет поток активно крутиться на процессоре, а блокирует его до освобождения блокировки.

## Идея реализации

Реализация построена на `std::atomic<uint32_t>` и `atomic::wait / notify_one`.

Используются три состояния мьютекса:

- `0` — свободен;
- `1` — захвачен, ожидающих потоков нет;
- `2` — захвачен, есть ожидающие потоки.

### Логика `Lock`

Если мьютекс свободен, поток быстро захватывает его через CAS без перехода в ядро.

Если мьютекс уже занят, состояние переводится в режим `есть ожидающие`, после чего поток блокируется через `wait`.

После пробуждения поток повторяет попытку захвата.

### Логика `Unlock`

При освобождении мьютекса состояние переводится в `свободен`.

Если известно, что были ожидающие потоки, вызывается `notify_one`, чтобы разбудить один из них.

## Тестирование

Проверяются следующие сценарии:

- простой захват и освобождение;
- повторный `lock/unlock`;
- независимость разных мьютексов;
- взаимное исключение;
- блокирующее ожидание при конкуренции.

Реализация показывает, как можно построить mutex на основе атомика и futex-подобного механизма `atomic::wait`.
