# MyCPU FUSE Project

## Описание

Данный проект представляет собой эмулятор вычислительного процессора в виде драйвера на базе FUSE.

После запуска драйвер монтирует дерево устройств следующего вида:

/dev/mycpu/
- ctrl
- unit0/
  - pram
  - lram
- unit1/
  - pram
  - lram
- ...
- unitN/
  - pram
  - lram

Количество устройств `unit` задается при запуске программы через параметр `--units` и во время работы не изменяется.

## Назначение файлов устройств

### ctrl

Файл `ctrl` используется для запуска выбранного юнита.

Если записать в `ctrl` номер устройства, соответствующий `unit` будет выполнен.  
При чтении `ctrl` возвращает список выполненных запусков.

Пример:

echo 3 > /dev/mycpu/ctrl
cat /dev/mycpu/ctrl

### lram

`lram` — локальная память устройства, в которую можно записывать входные данные и из которой можно читать результат вычислений.

### pram

`pram` — память программы устройства.  
В нее записывается описание операции, которую нужно выполнить над данными из `lram`.

## Краткая архитектура

Проект состоит из трех основных частей.

### 1. FUSE-слой

Файл `cpuemufs.py` реализует файловую систему FUSE и отображает внутреннюю модель устройства в виде дерева директорий и файлов.

### 2. Модель устройства

Файл `device.py` описывает:
- локальную память `Lram`;
- программную память `Pram`;
- вычислительный блок `Unit`;
- контроллер `Ctrl`;
- устройство `Device`.

### 3. Точка запуска

Файл `cpuemu` запускает FUSE-драйвер, принимает аргументы командной строки и монтирует файловую систему.

## Пример использования

1. Смонтировать устройство:
   ./cpuemu --units=4 /tmp/mycpu

2. Записать данные в память:
   cat testdata.bin > /tmp/mycpu/unit0/lram

3. Записать программу:
   echo "[u16:200:400]add(u8:0, u8:100)" > /tmp/mycpu/unit0/pram

4. Запустить устройство:
   echo 0 > /tmp/mycpu/ctrl

5. Прочитать результат:
   cat /tmp/mycpu/unit0/lram

## Тестирование

В проекте предусмотрены два типа тестов:

### Unit tests

Базовые unit-тесты запускаются через Catch2.

### Integration tests

Интеграционные тесты находятся в директории `integration/`:
- `tree-test.sh` — проверяет структуру дерева устройств;
- `device-test.sh` — проверяет работу `ctrl`, `lram` и `pram`.

## Сборка

Сборка проекта выполняется через CMake:

cmake -S . -B build
cmake --build build

## Документация

Проект поддерживает генерацию документации через Doxygen.

Если Doxygen установлен, документацию можно собрать командой:

cmake --build build --target doc

## Установка

В проекте подготовлена базовая поддержка установки и упаковки через CPack.

Установка:

cmake --install build

Создание пакета:

cpack --config build/CPackConfig.cmake

## Ограничения текущего этапа

На данном этапе проект является учебным и развивается поэтапно.  
Это означает, что не все функциональные требования обязаны быть полностью реализованы сразу.

Основной фокус текущей версии:
- корректная структура дерева устройств;
- работа файлов `ctrl`, `lram`, `pram`;
- возможность тестирования;
- поддержка документации и сборки.

## Вывод

Проект демонстрирует подход к эмуляции вычислительного устройства через FUSE, когда взаимодействие с устройством выполняется через привычную файловую модель Linux.
