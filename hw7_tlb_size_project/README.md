# TLB Size Project

## Описание

В данной работе проводится серия экспериментов для оценки размера TLB по изменению времени доступа к памяти.

TLB (Translation Lookaside Buffer) — это кэш трансляций виртуальных адресов в физические.  
Если рабочий набор страниц помещается в TLB, доступ к памяти выполняется быстрее.  
Когда число используемых страниц превышает емкость TLB, появляются дополнительные промахи, и среднее время доступа возрастает.

## Цель эксперимента

Цель — оценить размер TLB экспериментально, не используя `cpuid`, CPU-Z и другие внешние средства определения характеристик процессора.

## Идея измерения

Используется страничный паттерн доступа: программа обращается к одному элементу на каждой странице памяти.

Это позволяет минимизировать влияние обычных кэшей данных и сильнее проявить влияние именно TLB.

### Основной паттерн

Для массива памяти выбирается шаг, равный размеру страницы:

- обычно 4096 байт;
- для `int`, это шаг `4096 / sizeof(int)` элементов.

Затем выполняется обход массива по страницам и измеряется среднее время доступа на одну страницу.

При постепенном росте количества страниц можно увидеть момент, когда время заметно увеличивается.  
Этот скачок обычно соответствует выходу рабочего набора за пределы TLB.

## Что измеряется

Для разных значений числа страниц измеряется:

- общее время серии проходов;
- среднее время доступа на одну страницу;
- число страниц в рабочем наборе.

Результаты сохраняются в CSV-файл для последующего анализа и построения графика.

## Как использовать

### Сборка

cmake -S . -B build  
cmake --build build

### Запуск эксперимента

./build/tlb_bench

Можно указать имя выходного файла:

./build/tlb_bench data/results.csv

## Формат результата

Программа сохраняет CSV с колонками:

pages,time_ns_per_access

Это позволяет легко построить график зависимости среднего времени доступа от числа страниц.

## Пример интерпретации

Если при росте числа страниц до 64 время почти не меняется, а после 64–128 страниц заметно увеличивается, можно сделать вывод, что размер TLB находится примерно в этом диапазоне.

## Вывод

Проект демонстрирует способ экспериментальной оценки размера TLB через измерение времени доступа к памяти при страничном паттерне обхода.  
Основная идея состоит в том, что увеличение рабочего набора страниц сверх емкости TLB приводит к росту времени доступа.
