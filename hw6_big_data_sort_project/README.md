# Big Data Sort Project

## Описание

Данный проект решает задачу сортировки большого бинарного файла с 64-битными целыми числами по возрастанию при ограничении на объем используемой виртуальной памяти.

Числа хранятся не в текстовом виде, а как последовательность значений типа `int64_t`.

Программа поддерживает два режима запуска:

sort_data <file_path>  
sort_data <file_path> <memory_limit_mb>

Если лимит памяти не задан, он вычисляется автоматически как `1/10` от размера файла.

## Идея решения

Используется внешний merge sort:

1. файл разбивается на блоки, которые помещаются в заданный лимит памяти;
2. каждый блок отображается в память, сортируется и записывается во временный файл;
3. затем все отсортированные блоки сливаются в один итоговый файл;
4. результат заменяет исходный файл.

Для работы с большими файлами используется отображение файла в память:
- `mmap` на Linux/macOS.

## Основные компоненты

### sort_data

Главная программа сортировки.

Принимает путь к бинарному файлу и опционально лимит памяти в мегабайтах.

### generate_data

Утилита генерации тестового бинарного файла со случайными `int64_t`.

Пример:

generate_data data.bin 1000000

Это создаст файл `data.bin` с одним миллионом чисел.

### verify_sorted

Утилита проверки результата сортировки.

Пример:

verify_sorted data.bin

Если файл отсортирован корректно, программа завершится успешно.

## Алгоритм

### Шаг 1. Разбиение на чанки

Исходный файл читается порциями, размер которых не превышает заданный лимит памяти.

Каждый чанк:
- отображается в память;
- копируется в обычный буфер;
- сортируется через `std::sort`;
- записывается во временный файл.

### Шаг 2. Многопутевое слияние

Все временные файлы открываются на чтение.  
Из каждого файла берется текущее минимальное значение, после чего выполняется k-way merge через min-heap.

Результат записывается в новый временный файл, который затем заменяет исходный.

## Ограничение памяти

В данной учебной реализации лимит памяти контролируется на уровне:
- размера входного чанка;
- буфера на слиянии.

Если лимит не задан, используется значение:

memory_limit = max(file_size / 10, sizeof(int64_t))

## Сборка

cmake -S . -B build  
cmake --build build

## Использование

### Сортировка файла

./build/sort_data data.bin  
./build/sort_data data.bin 128

### Генерация файла

./build/generate_data data.bin 100000

### Проверка сортировки

./build/verify_sorted data.bin

## Ограничения

Проект ориентирован на Linux/macOS и использует `mmap`.

В текущей реализации:
- временные файлы создаются рядом с исходным;
- merge выполняется однопоточно;
- ожидается, что размер файла кратен `sizeof(int64_t)`.

## Вывод

Проект демонстрирует практический подход к сортировке данных, которые не помещаются целиком в память.  
Основная идея состоит в разбиении файла на отсортированные куски и их последующем слиянии.
