# Future and Promise

## Описание

В данной работе реализована простая блокирующая пара `Future<T>` и `Promise<T>`.

`Promise<T>` используется потоком-producer для передачи результата:
- значения через `SetValue(...)`;
- исключения через `SetException(...)`.

`Future<T>` используется потоком-consumer для получения результата через `Get()`.

Если результат еще не готов, вызов `Get()` блокирует поток до тех пор, пока `Promise` не передаст значение или исключение.

## Идея реализации

`Future` и `Promise` разделяют общее состояние через `std::shared_ptr`.

Во внутреннем состоянии хранятся:
- мьютекс;
- условная переменная;
- флаг готовности результата;
- `std::variant<T, std::exception_ptr>` для хранения либо значения, либо исключения.

Когда `Promise` вызывает `SetValue` или `SetException`, результат записывается в общее состояние, после чего ожидающий поток пробуждается.

Когда `Future` вызывает `Get`, он ждет, пока результат не станет готов, а затем:
- возвращает значение;
- либо повторно выбрасывает исключение.

## Тестирование

Проверяются следующие сценарии:
- получение обычного значения;
- передача исключения;
- ожидание результата из другого потока;
- разрушение `Future` до передачи значения;
- разрушение `Promise` после передачи значения;
- работа нескольких futures одновременно.

build\future_tests.exe

## Ограничения

Данная реализация является учебной и упрощенной:
- `SetValue`, `SetException` и `Get` предполагаются одноразовыми;
- нарушение этого контракта считается некорректным использованием;
- вариант `void` не рассматривается, вместо этого при необходимости можно использовать `std::monostate`.

## Вывод

В ходе работы реализована простая модель `Future/Promise`, позволяющая безопасно передавать результат вычисления между потоками и синхронизировать producer и consumer через общее состояние.
